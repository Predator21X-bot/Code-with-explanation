# 1. Array Representation of Binary Tree

A complete binary tree can be stored efficiently using an array.

Consider:

```text
          10
        /    \
      20      30
     /  \    /  \
   40   50 60   70
```

Array Representation:

```text
Index : 1  2  3  4  5  6  7

Value :10 20 30 40 50 60 70
```

---

## Index Relationships

For a node at index `i`:

### Left Child

```text
2i
```

### Right Child

```text
2i + 1
```

### Parent

```text
⌊i/2⌋
```

Example:

```text
Node at index 3 = 30

Left Child  = 6
Right Child = 7
Parent      = 1
```

These formulas work because a heap is stored as a complete binary tree.

---

# 2. Complete Binary Tree

A Complete Binary Tree is:

```text
All Levels Completely Filled

Except Possibly Last Level

Last Level Filled
From Left To Right
```

Valid:

```text
        10
      /    \
     20    30
    / \
   40 50
```

Invalid:

```text
        10
      /    \
     20    30
           /
          40
```

A heap must always be a complete binary tree.

---

# 3. Heap

A Heap is a Complete Binary Tree that satisfies a Heap Property.

---

## Max Heap

Parent is always greater than children.

```text
         50
       /    \
      30     40
     / \    / \
   10 20 15 25
```

Condition:

```text
Parent ≥ Children
```

---

## Min Heap

Parent is always smaller than children.

```text
         10
       /    \
      20     30
     / \    / \
   40 50 60 70
```

Condition:

```text
Parent ≤ Children
```

---

# Heap Characteristics

A Heap is:

```text
Complete Binary Tree
+
Heap Property
```

Important:

```text
Heap is NOT Sorted
```

Only parent-child ordering is guaranteed.

---

# 4. Insertion in Max Heap

Suppose heap:

```text
       50
      /  \
    30    40
```

Insert:

```text
45
```

---

## Step 1

Insert at next available position.

```text
       50
      /  \
    30    40
   /
 45
```

---

## Step 2

Compare with parent.

```text
45 > 30
```

Swap.

```text
       50
      /  \
    45    40
   /
 30
```

---

## Step 3

Compare again.

```text
45 < 50
```

Stop.

---

### Insert Algorithm

```cpp
void Insert(int H[], int n)
{
    int i = n;
    int temp = H[i];

    while(i > 1 && temp > H[i/2])
    {
        H[i] = H[i/2];
        i = i/2;
    }

    H[i] = temp;
}
```

---

# Complexity of Insertion

Tree height:

```text
log n
```

Therefore:

```text
Insertion = O(log n)
```

---

# 5. Deletion from Max Heap

Delete always removes:

```text
Root Element
```

Largest element.

---

Example:

```text
        50
      /    \
    45      40
   /  \
 30   20
```

Delete:

```text
50
```

---

## Step 1

Replace root with last element.

```text
        20
      /    \
    45      40
   /
 30
```

---

## Step 2

Compare with larger child.

```text
45 > 40
```

Swap.

```text
        45
      /    \
    20      40
   /
 30
```

---

## Step 3

Continue downward.

```text
30 > 20
```

Swap.

```text
        45
      /    \
    30      40
   /
 20
```

Heap restored.

---

### Complexity

```text
O(log n)
```

# 6. Heap Sort

Heap Sort uses:

```text
Repeated Heap Deletion
```

---

Given:

```text
10 20 30 25 5 40 35
```

---

## Step 1

Create Max Heap.

```text
         40
       /    \
     25      35
    /  \    / \
  10  5  30 20
```

---

## Step 2

Delete root repeatedly.

Elements removed:

```text
40
35
30
25
20
10
5
```

---

## Sorted Result

Ascending order:

```text
5 10 20 25 30 35 40
```

---

## Complexity

Building heap:

```text
O(n log n)
```

Deleting all elements:

```text
O(n log n)
```

Total:

```text
O(n log n)
```

---

# 7. Heapify

Heapify converts an array into a heap.

Example:

```text
10 20 30 25 5 40 35
```

Not a heap.

---

After Heapify:

```text
40 25 35 10 5 30 20
```

Max Heap formed.

---

## Heapify Idea

Start from:

```text
Last Non-Leaf Node
```

Move upward.

Restore heap property at each node.

---

### Complexity

```text
O(n)
```

This is more efficient than inserting elements one by one.

# 8. Priority Queue

A Priority Queue is an ADT where:

```text
Highest Priority Item
Removed First
```

Not:

```text
FIFO
```

Not:

```text
LIFO
```

---

## Heap as Priority Queue

Max Heap:

```text
Largest Element
Highest Priority
```

Min Heap:

```text
Smallest Element
Highest Priority
```

---

## Operations

### Insert

```text
O(log n)
```

### Delete

```text
O(log n)
```

### Find Max/Min

```text
O(1)
```

---

# Complete C++ Insert Example

```cpp
#include <iostream>
using namespace std;

void Insert(int H[], int n)
{
    int i = n;
    int temp = H[n];

    while(i > 1 && temp > H[i/2])
    {
        H[i] = H[i/2];
        i = i/2;
    }

    H[i] = temp;
}

int main()
{
    int H[20] = {0,10,20,30,25,5,40,35};

    for(int i=2;i<=7;i++)
        Insert(H,i);

    return 0;
}
```

# Key Takeaway

A Heap is a complete binary tree stored efficiently in an array.

```text
Complete Binary Tree
            +
Heap Property
            =
Heap
```

Using insertion and deletion operations on heaps allows efficient implementation of:

```text
Heap Sort
Priority Queues
Scheduling Systems
Graph Algorithms
```

which is why heaps are one of the most important tree-based data structures in algorithms.
