# 14. Two Way MergeSort (Iterative Method)

## Overview

Two-Way Merge Sort is an **iterative sorting algorithm** that repeatedly merges adjacent sorted subarrays until the entire array becomes sorted.

Before understanding Merge Sort, it is important to understand:

1. Merging
2. M-Way Merging
3. Merge Pattern
4. Two-Way Merge Sort (Iterative)

---

# 1. Merging

Merging is the process of combining two sorted lists into a single sorted list.

### Example

```text
List 1: 2 8 15 18
List 2: 4 7 10 20
```

Merged Result:

```text
2 4 7 8 10 15 18 20
```

### Algorithm

1. Compare first elements of both lists.
2. Copy the smaller element to the output array.
3. Move the corresponding pointer.
4. Repeat until one list is exhausted.
5. Copy remaining elements.

### Time Complexity

For two lists of sizes `m` and `n`:

```text
O(m + n)
```

---

# 2. M-Way Merging

Instead of merging only two sorted lists, multiple sorted lists can be merged together.

### Example

```text
L1 = 2 8 15
L2 = 4 7 20
L3 = 1 5 30
```

Merged Result:

```text
1 2 4 5 7 8 15 20 30
```

### Applications

- External Sorting
- Database Systems
- File Systems
- Large Data Processing

# 4. Idea of Two-Way Merge Sort

Consider:

```text
8 3 7 4 9 2 6 5
```

Initially every element is treated as a sorted list of size 1.

```text
[8] [3] [7] [4] [9] [2] [6] [5]
```

---

# 5. First Pass

Merge adjacent lists.

```text
[8] [3] -> [3 8]
[7] [4] -> [4 7]
[9] [2] -> [2 9]
[6] [5] -> [5 6]
```

Result:

```text
[3 8] [4 7] [2 9] [5 6]
```

Subarray Size:

```text
2
```

---

# 6. Second Pass

Merge lists of size 2.

```text
[3 8] [4 7]
```

↓

```text
[3 4 7 8]
```

```text
[2 9] [5 6]
```

↓

```text
[2 5 6 9]
```

Result:

```text
[3 4 7 8]
[2 5 6 9]
```

Subarray Size:

```text
4
```

---

# 7. Third Pass

Merge lists of size 4.

```text
[3 4 7 8]
[2 5 6 9]
```

↓

```text
[2 3 4 5 6 7 8 9]
```

Array becomes sorted.

---

# 8. Observation

After every pass:

```text
1 → 2 → 4 → 8 → 16 ...
```

Subarray size doubles.

Number of passes:

```text
log₂(n)
```

# 9. Iterative Merge Sort Algorithm

```cpp
for (int size = 1; size < n; size *= 2)
{
    merge adjacent subarrays of size 'size';
}
```

# 10. Merge Function

## C++ Implementation

```cpp
#include <iostream>
using namespace std;

void Merge(int A[], int l, int mid, int h)
{
    int i = l;
    int j = mid + 1;
    int k = l;

    int B[100];

    while (i <= mid && j <= h)
    {
        if (A[i] < A[j])
            B[k++] = A[i++];
        else
            B[k++] = A[j++];
    }

    while (i <= mid)
        B[k++] = A[i++];

    while (j <= h)
        B[k++] = A[j++];

    for (int p = l; p <= h; p++)
        A[p] = B[p];
}
```

# 11. Iterative Merge Sort (Abdul Bari Style)

```cpp
#include <iostream>
using namespace std;

void Merge(int A[], int l, int mid, int h)
{
    int i = l;
    int j = mid + 1;
    int k = l;

    int B[100];

    while (i <= mid && j <= h)
    {
        if (A[i] < A[j])
            B[k++] = A[i++];
        else
            B[k++] = A[j++];
    }

    while (i <= mid)
        B[k++] = A[i++];

    while (j <= h)
        B[k++] = A[j++];

    for (int p = l; p <= h; p++)
        A[p] = B[p];
}

void IterativeMergeSort(int A[], int n)
{
    int p, l, h, mid, i;

    for (p = 2; p <= n; p = p * 2)
    {
        for (i = 0; i + p - 1 < n; i += p)
        {
            l = i;
            h = i + p - 1;
            mid = (l + h) / 2;

            Merge(A, l, mid, h);
        }
    }

    if (p / 2 < n)
    {
        Merge(A, 0, p / 2 - 1, n - 1);
    }
}

int main()
{
    int A[] = {8, 3, 7, 4, 9, 2, 6, 5};

    int n = sizeof(A) / sizeof(A[0]);

    IterativeMergeSort(A, n);

    cout << "Sorted Array:\n";

    for (int i = 0; i < n; i++)
        cout << A[i] << " ";

    return 0;
}
```

# 13. Complexity Analysis

## Time Complexity

Each pass processes all elements:

```text
O(n)
```

Number of passes:

```text
O(log n)
```

Therefore:

```text
O(n log n)
```

---

## Space Complexity

Temporary auxiliary array is required.

```text
O(n)
```

---

# 14. Properties of Two-Way Merge Sort

| Property | Value |
|-----------|--------|
| Technique | Iterative |
| Stable | Yes |
| In-Place | No |
| Best Case | O(n log n) |
| Average Case | O(n log n) |
| Worst Case | O(n log n) |
| Extra Space | O(n) |

# 15. Important Interview Questions

### Why is Merge Sort Stable?

Because equal elements preserve their relative order during merging.

---

### Why is Merge Sort not In-Place?

Because it requires an auxiliary array for merging.

---

### Number of Passes Required?

```text
log₂(n)
```

---

### Complexity of Merging Two Sorted Lists?

```text
O(m + n)
```

# Key Takeaways

- Merging combines two sorted arrays.
- Two-Way Merge Sort repeatedly merges adjacent sorted blocks.
- Initially each element is considered a sorted list of size 1.
- Subarray size doubles after every pass.
- Number of passes = `log₂(n)`.
- Time Complexity = `O(n log n)`.
- Auxiliary Space = `O(n)`.
- Stable sorting algorithm.
- Iterative version of Merge Sort.
