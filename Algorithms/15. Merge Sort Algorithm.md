# 15. Merge Sort Algorithm (Recursive Divide & Conquer)

## Overview

Merge Sort is a **Divide and Conquer** sorting algorithm.

The algorithm works in three steps:

1. **Divide** the array into two halves.
2. **Recursively sort** both halves.
3. **Merge** the sorted halves.

Unlike the previous lecture (**Two-Way Merge Sort - Iterative Method**), Merge Sort uses **recursion**.

---

# Divide and Conquer Strategy

Merge Sort follows:

```text
Divide
   ↓
Divide
   ↓
Divide
   ↓
Merge
   ↑
Merge
   ↑
Merge
```

The array is repeatedly divided until each subarray contains only one element.

Since a single element is already sorted, merging begins from the bottom.

---

# Merge Sort Algorithm

## Recursive Function

```cpp
MergeSort(A, l, h)
{
    if(l < h)
    {
        mid = (l + h) / 2;

        MergeSort(A, l, mid);

        MergeSort(A, mid + 1, h);

        Merge(A, l, mid, h);
    }
}
```

# Merge Function

The Merge function combines two sorted subarrays into a single sorted array.

## Example

```text
Left  = 3 5 7
Right = 2 4 8
```

Output:

```text
2 3 4 5 7 8
```

---

# C++ Implementation

## Merge Function

```cpp
#include <iostream>
using namespace std;

void Merge(int A[], int l, int mid, int h)
{
    int i = l;
    int j = mid + 1;
    int k = l;

    int B[100];

    while(i <= mid && j <= h)
    {
        if(A[i] < A[j])
            B[k++] = A[i++];
        else
            B[k++] = A[j++];
    }

    while(i <= mid)
        B[k++] = A[i++];

    while(j <= h)
        B[k++] = A[j++];

    for(int p = l; p <= h; p++)
        A[p] = B[p];
}
```

## Complete Program

```cpp
#include <iostream>
using namespace std;

void Merge(int A[], int l, int mid, int h)
{
    int i = l;
    int j = mid + 1;
    int k = l;

    int B[100];

    while(i <= mid && j <= h)
    {
        if(A[i] < A[j])
            B[k++] = A[i++];
        else
            B[k++] = A[j++];
    }

    while(i <= mid)
        B[k++] = A[i++];

    while(j <= h)
        B[k++] = A[j++];

    for(int p = l; p <= h; p++)
        A[p] = B[p];
}

void MergeSort(int A[], int l, int h)
{
    if(l < h)
    {
        int mid = (l + h) / 2;

        MergeSort(A, l, mid);

        MergeSort(A, mid + 1, h);

        Merge(A, l, mid, h);
    }
}

int main()
{
    int A[] = {3,7,5,6,4,8,2};

    int n = sizeof(A)/sizeof(A[0]);

    MergeSort(A, 0, n - 1);

    cout << "Sorted Array:\n";

    for(int i = 0; i < n; i++)
        cout << A[i] << " ";

    return 0;
}
```

# Time Complexity Analysis

## Division Phase

Array is divided recursively.

Number of levels:

```text
log₂(n)
```

---

## Merge Phase

At every level, all elements participate in merging.

Work per level:

```text
O(n)
```

---

## Total Complexity

```text
O(n) × O(log n)
```

Therefore:

```text
O(n log n)
```

---

# Recurrence Relation

```text
T(n) = 2T(n/2) + O(n)
```

Using Master Theorem:

```text
T(n) = O(n log n)
```

---

# Space Complexity

Merge Sort requires an auxiliary array.

```text
O(n)
```

Recursion stack:

```text
O(log n)
```

Overall:

```text
O(n)
```

---

# Why Merge Sort is Stable?

Consider:

```text
5a 3 5b 2
```

After sorting:

```text
2 3 5a 5b
```

Relative order of equal elements is preserved.

Hence:

```text
Stable Sorting Algorithm
```

# Properties of Merge Sort

| Property     | Value            |
| ------------ | ---------------- |
| Technique    | Divide & Conquer |
| Recursive    | Yes              |
| Stable       | Yes              |
| In-Place     | No               |
| Best Case    | O(n log n)       |
| Average Case | O(n log n)       |
| Worst Case   | O(n log n)       |
| Extra Space  | O(n)             |

# Iterative Merge Sort vs Recursive Merge Sort

| Feature           | Iterative  | Recursive  |
| ----------------- | ---------- | ---------- |
| Approach          | Bottom-Up  | Top-Down   |
| Uses Recursion    | No         | Yes        |
| Extra Stack Space | No         | Yes        |
| Complexity        | O(n log n) | O(n log n) |
| Stability         | Stable     | Stable     |

# Key Takeaways

* Merge Sort follows the **Divide and Conquer** paradigm.
* Array is recursively divided until subarrays contain one element.
* Merging starts from the lowest level.
* Recurrence relation:

```text
T(n) = 2T(n/2) + O(n)
```

* Time Complexity:

```text
O(n log n)
```

* Auxiliary Space:

```text
O(n)
```

* Stable sorting algorithm.
* Performs consistently in all cases.
* One of the most important sorting algorithms used in practice.
